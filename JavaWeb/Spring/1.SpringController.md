# Spring Controller
개발을 진행하면서 핵심 로직을 작성하는 component.

## Annotation 기반
- 실무에서 가장 많이 쓰이는 code style.
- Dispatcher Servlet 또한 해당 방식의 Mapper를 먼저 사용한다.
```java
@Controller
@RequestMapping("/springmvc/members")  // 1. URI의 루트 경로를 class에 작성할 수 있다.
public class SpringMemberController {
    
    private final MemberRepository memberRepository = MemberRepository.getInstance();

    @RequestMapping(value = "/new-form", method = RequestMethod.GET)  // 2. RequestMapping 사용시 method, value를 둘다 작성한다.
    public String newForm() {
        return "new-form";
    }

    @PostMapping("save-result")  // 3. PostMapping 사용시 URI만 작성한다.
    public String save(
        @RequestParam("username") Stinrg username,  // 4. RequestParam annotation을 사용하면 request로부터 특정 변수를 가져올 수 있다.
        @RequestParam("age") int age,
        Model model  // 5. Model을 인자로 넘겨주면 dispatcher servlet에서 처리하도록 반영할 수 있다.
    ) {
        Member member = new Member(username, age);
        memberRepository.save(member);
        
        model.addAttribute("member", member);  // 6. model에 key-value 쌍을 추가하는 method.

        return "save-result"
    }

    @GetMapping
    public ModelAndView members() {  // 7. String 말고 ModelAndView 객체로 반환할 수 있다.
        List<Member> members = memberRepository.findAll();

        ModelAndView mv = new ModelAndView("members");
        mv.addObject("members", members);  // 8. ModelAndView 객체의 모델에 key-value 쌍을 추가하는 method.

        return mv;
    }

}
```
